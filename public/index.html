나의 비전

저는 세상에 기여하는 개발자가 되고 싶습니다. 기술과 프로그래밍 언어는 정말 다양하지만, 결국 우리가 해야 할 일은 사람들에게 더 나은 편리함과 경험을 제공하는 것이라고 생각합니다.

개발자는 단순히 코드를 작성하는 사람이 아닙니다. 사용자들이 겪는 문제를 이해하고, 그 문제를 해결하는 역할을 맡고 있습니다. 기술은 그저 도구일 뿐이고, 진정한 가치는 그 도구를 통해 얼마나 많은 사람들의 삶을 긍정적으로 변화시킬 수 있는가에 있습니다.

그래서 저는 기술을 활용해 사람들이 일상에서 느끼는 불편함을 줄이고, 그들의 삶을 더욱 풍요롭게 만드는 데 기여하고자 합니다. 혁신적인 아이디어와 솔루션으로 사람들의 삶을 더 쉽고 즐겁게 만들어가는 개발자가 되기 위해 계속해서 노력할 것입니다.

저의 비전은 기술을 다루는 것을 넘어, 그 기술이 어떻게 사람들의 삶에 실질적인 가치를 더할 수 있는지를 탐구하는 것입니다.

Project Info

기존에 구현한 프론트엔드 기반의 프로젝트를 배포하기 위해 AWS를 사용하여 배포하였습니다.

기존에 구현한 여러 프론트엔드 기반 프로젝트에서 API를 사용하는 경우가 많아, 프론트엔드 페이지에 API 키가 노출될 위험을 피하기 위해 EC2 서버에 Node.js 환경을 구성하였습니다.

이를 통해 Node.js 프로젝트로 전환하고 API 키 설정을 추가했습니다. Docker를 활용하여 Node.js를 기반으로 한 컨테이너를 생성한 후, 이를 서버에 업로드하여 프로젝트를 배포하였습니다.

톰캣서버를 사용하는 JSP기반 프로젝트 은하수책방을 배포하기 위해 카페24 호스팅을 지원받아 해당 호스팅 서버에 프로젝트를 배포하고, 포트폴리오 페이지에 링크를 추가하였습니다.

프로젝트 개요: 은하수 책방

은하수 책방은 프론트엔드부터 데이터베이스(DB) 연결까지 풀스택으로 개발한 3인 공동 프로젝트입니다. 이 프로젝트는 도서 정보를 제공하고 판매하는 쇼핑몰 페이지로, 사용자들이 더욱 몰입할 수 있도록 책에 대한 커뮤니티 기능을 추가했습니다.

개발 과정

각각의 페이지는 HTML, SCSS, JS를 기반으로 디자인을 완성한 후, Java, Servlet, JSP를 사용하여 백엔드 개발과 DB 제어를 진행했습니다. 처음 진행하는 협업 프로젝트인 만큼, GitHub에서 형상 관리 방법을 배우고, 프로젝트 설정에 따른 충돌 문제를 해결하는 등 다양한 경험을 쌓았습니다.

톰캣 서버를 이용하여 카페24 호스팅을 통해 배포하였으며, 배포 과정에서 발생한 버전 충돌 문제와 카페24의 서블릿 매핑 지정 문제를 해결하며 현업의 다양한 문제를 마주했습니다. 이러한 과정을 통해 개발자로서 성장하는 기회를 가졌습니다.

기술적 성과

실제 DB에서 데이터를 관리하고 화면에 반영하면서 백엔드 구조, 서버, WAS, DB와 브라우저 간의 연동 방식에 대해 깊이 고찰하는 계기가 되었습니다. 프로젝트에서 메인화면과 개인화면 개발을 맡았으며, 모든 페이지에 적용되는 헤더 컴포넌트에서 DB 검색 기능을 구현했습니다. 이를 통해 페이지 이동과 DB 조작 외에도 서블릿을 이용한 자체 DB 검색 API 개발 경험을 쌓았습니다. 이 과정에서 Java와 JS 간의 차이점, 데이터 양식 및 각 언어에서 데이터 처리 방식에 대해 심도 있게 연구할 수 있었습니다.

프론트엔드에서 백엔드로 전환하는 과정에서 기초적인 DB 테이블 구조와 칼럼 구조를 설계하며 데이터 처리 방식과 효율적인 관리 방법에 대해 고민했습니다. 또한, 각 페이지의 Java 패키지를 생성하고 JSP 파일을 WEB-INF 디렉터리에 위치시켜 보안성을 강화했습니다. 이 과정에서 서블릿을 통해 URI 요청을 분석하고, 각 페이지에 접근하도록 포워딩하는 경험은 추후 검색 기능 개발 시 보다 간결한 데이터 전송 구조를 구축하는 데 중요한 발판이 되었습니다.

마지막으로, SCSS와 JSP를 이용해 페이지의 일부분을 컴포넌트화하여 유지보수성을 향상시키는 경험은 효과적인 페이지 구성에 대한 이해를 더욱 깊게 해주었습니다.

이 프로젝트를 통해 얻은 다양한 경험과 지식은 향후 개발자로서의 성장에 큰 도움이 될 것입니다.

사용 기술

HTML: 웹 페이지의 구조를 위해 HTML5를 사용하였으며, 도서 정보와 이미지, 버튼 등을 효율적으로 배치하여 사용자 인터페이스를 구현했습니다.

SCSS: 스타일링의 효율성을 높이기 위해 SCSS를 활용하였습니다. 변수, 중첩, 믹스인 등을 사용하여 코드의 재사용성을 극대화하고 유지보수성을 향상시켰습니다.

JavaScript: 사용자와의 상호작용을 위한 동적인 기능 구현에 JavaScript를 사용했습니다. 예를 들어, 도서 검색 기능과 페이지 전환 효과를 추가하여 사용자 경험을 개선했습니다.

Java: 백엔드 로직과 데이터베이스 연결을 위해 Java를 사용하였습니다. Servlet과 JSP를 통해 서버 측 처리를 수행하며, 클라이언트와의 데이터 전송을 효율적으로 관리하였습니다.

JSP: 동적인 웹 콘텐츠 생성을 위해 JSP를 활용했습니다. JSP를 통해 데이터베이스에서 가져온 정보를 사용자에게 표시하며, HTML과 Java 코드를 결합하여 효율적인 페이지 렌더링을 구현했습니다.

Servlet: 클라이언트의 요청을 처리하고 응답을 생성하기 위해 서블릿을 사용했습니다. 서블릿을 통해 요청 매핑과 세션 관리를 효율적으로 수행하고, 데이터베이스와의 연결을 통해 동적인 웹 기능을 제공하였습니다.

사용 라이브러리

Gson: JSON 데이터를 처리하기 위해 Gson 라이브러리를 사용했습니다. header 섹션에서 페이지 전환 없이 요소를 생성하기 위해 fetch 요청으로 서블릿에 요청을 보냈습니다. 서블릿 요청 처리 과정에서 생성된 bean 형태의 데이터를 프론트엔드의 header에게 효과적으로 전달하기 위해 Gson 라이브러리를 도입했습니다. 이를 통해 각각의 데이터 형태를 효과적으로 컨트롤할 수 있었습니다.

Jakarta Servlet API: 서블릿 기반의 웹 애플리케이션 개발을 위해 Jakarta Servlet API를 활용했습니다. 실제 개발 과정에서는 Tomcat 9에서 개발하였으나, 배포 시에는 Tomcat 10을 사용했습니다. 이로 인해 HTTP 요청 서블릿을 구동하기 위해 추가적인 설정이 필요했습니다. 이 라이브러리를 통해 HTTP 요청 및 응답을 효과적으로 처리하고, 웹 애플리케이션의 구조를 체계적으로 구성할 수 있었습니다.

작업과정

"오늘 뭐하지?" 프로젝트는 API를 활용하여 데이터를 전송하고, 수신된 데이터를 가공하여 화면의 다양한 요소를 구성하는 데 중점을 둔 웹 애플리케이션입니다. 이 프로젝트는 원페이지 슬라이드 화면으로 설계되어 있어 추가적인 로딩 없이 매끄러운 화면 전환을 구현했습니다. 각 요소의 스타일을 조정하여 사용자에게 자연스러운 경험을 제공하고, 화면 전환 시 부자연스러운 부분이 없도록 하였습니다.

실시간 데이터를 제공받아 사용자가 직접 활용할 수 있는 구성으로 설계하였으며, 프론트엔드만으로 개발된 프로젝트이기 때문에 데이터베이스(DB)를 사용할 수 없는 상황에서 사용자 정보를 저장하지 못하는 문제를 로컬스토리지를 통해 어느 정도 해결하였습니다. 이를 통해 사용자 맞춤형 경험을 강화했습니다.

첫 번째 화면에서는 전체적인 날씨 및 기후 상황을 시각적으로 표시하여, 폭설이나 폭우와 같은 기상 조건, 바람의 세기, 상대 기온 등이 가장 높거나 낮은 지역의 정보를 제공합니다. 이를 통해 사용자가 지역을 선택할 때 더 나은 편의성을 느낄 수 있도록 하였습니다.

두 번째 화면에서는 실제 등록된 지점의 좌표 정보를 활용하여 현재 날씨 및 교통 정보를 실시간으로 제공합니다. 도착 예상 시간대의 날씨 정보를 함께 보여줌으로써 사용자가 실제 나들이를 떠났을 때 유용하게 활용할 수 있는 정보를 제공합니다. 또한, 사용자가 자신의 현재 위치를 등록할 수 있도록 하여 개인 맞춤형 리스트를 구성할 수 있게 개발하였습니다.

세 번째 화면에서는 각 여행지에 대한 상대적인 평가 정보를 등록할 수 있는 기능을 추가하였습니다. 사용자는 주소 검색을 통해 원하는 지역을 리스트에 손쉽게 추가할 수 있습니다. 이러한 기능은 사용자 간의 정보 공유와 경험을 더욱 풍부하게 만들어줍니다.

각각의 화면에서 발생하는 정보를 로컬스토리지에 저장하고, 이 저장된 데이터를 바탕으로 새로운 화면을 구성하는 로직을 적용하여 보다 동적인 서비스를 제공할 수 있도록 개발하였습니다.

프로젝트를 진행하면서 느낀 점은 프론트엔드만으로 작업을 진행하다 보니, 외부에 공개되어서는 안 되는 데이터를 관리하는 .env 파일이나 데이터베이스의 부재가 서비스의 생동감을 제한하는 요소가 될 수 있다는 것이었습니다. 특히 API가 백엔드에서 요청을 통해 응답을 받도록 설정된 경우 등을 겪으면서, 실제 웹 개발에서는 모든 분야가 협력하여 하나의 완성된 서비스를 만들어야 한다는 깊은 깨달음을 얻었습니다. 이러한 경험은 앞으로의 개발 여정에 있어 중요한 밑거름이 될 것입니다.

기존 카카오톡의 개선 사항을 면밀히 분석하여 페이지를 리뉴얼하였습니다. 이 과정에서 카카오톡의 아이덴티티 컬러를 적극 활용하여 브랜드 이미지를 효과적으로 강조하였으며, 사용자에게 전달하고자 하는 내용을 간결하고 명확하게 배치하는 레이아웃을 설계하였습니다. 이를 통해 사용자는 필요한 정보를 쉽게 찾을 수 있도록 하였습니다.

본 프로젝트에서는 외부 라이브러리를 사용하지 않고, 모든 내용에 대한 HTML 구조와 CSS 스타일링을 직접 작성하였습니다. 특히, 시멘틱 웹을 추구하는 방향으로 작업하였으며, CSS에서는 여러 기본 선택자를 활용하여 SCSS 기술을 접목시켜 보다 효율적이고 가독성 높은 코드로 산출물을 제작하였습니다. 이러한 접근은 프로젝트의 유지 보수성과 확장성을 높이는 데 큰 도움이 되었습니다.

초기: 카카오 프로젝트를 진행하면서 새로운 트랜지션을 통한 페이지 애니메이션 효과 부여에 집중하여 개발하였습니다. 이 과정에서, 기존 채팅 화면의 기능을 구현하기 위해 `createElement` 메서드를 활용하여 새로운 DOM 요소를 생성하는 방법을 연구하였습니다. 더불어, 이러한 요소들이 자연스럽게 사용자와 상호작용할 수 있도록 다양한 관련 메서드의 기능을 탐구하였습니다. 이를 기반으로 사용자가 메시지를 입력했을 때, 자동으로 응답글이 생성되는 시스템을 구현하기 위해 `setTimeout` 메서드를 이용하여 지연 응답 기능을 추가했습니다.

또한, 화면에 존재하는 기존 데이터를 활용하여 새로운 요소 및 세션을 구성할 때, 어떻게 효율적으로 데이터를 불러올 수 있을지를 고민했습니다. 이 과정에서 `alt` 속성 등을 활용하여 이미지 저장 경로를 관리하고, 이를 통해 새로운 풀 프로필 화면이나 채팅 페이지를 불러올 때 미니 프로필 및 채팅 목록에서 경로를 불러와 이미지를 화면에 출력하는 로직을 설계하였습니다. 이로 인해 사용자 경험을 더욱 향상시킬 수 있었습니다.
